## Что это
Личный сайт-читалка для романов автора **Азата Туктарова**. Флипбук с переворачивающимися страницами, админка для загрузки книг, система закладок. Стек: Express + PostgreSQL + vanilla JS (без фреймворков).

**Книги:**
- «Тёмный Восход» (slug: `temny-voskhod`) — городское фэнтези/юмор: вампиры, ковбои, ангелы, коты-воины. Главы 1-3 реально загружены («Нетленка», «Кот и Бобёр», «Ангел Василий»).
- «Немая Рыбка из чужой реки» (slug предположительно `nemaya-rybka`) — пока без реального контента.

**Локальный путь:** `~/Documents/2026/творчество26/site/my-book-site` (⚠️ на компьютере пользователя случайно расплодились дубли папок `my-book-site2`...`5` от повторных распаковок — надо следить, чтобы редактировать только ОДНУ актуальную).

## Структура проекта
```
my-book-site/
├── server.js              Express + PostgreSQL, сессии, bcrypt, multer
├── db/schema.sql          users, books, chapters, reading_progress
├── lib/chapterSplitter.js Общая логика разбивки текста на главы
├── import-book.js         CLI загрузка книги (--folder | --file [--toc] [--preview])
├── import-docx.js         CLI загрузка одной главы
├── extract-toc.js         Авто-извлечение списка заголовков из .docx
└── public/
    ├── index.html         Главная: bento-hero (имя автора + "КНИГИ" + иллюстрация), каталог книг
    ├── book.html           Чтение книги (флипбук)
    ├── admin.html          Админка: книги/главы/массовая загрузка .docx
    ├── contents.html       Список глав книги + "Продолжить чтение"
    ├── registration.html   Вход/Регистрация (вкладки)
    ├── css/, js/, img/
```

## Ключевые решения и пофикшенные баги
1. **`DB_HOST=127.0.0.1`**, не `localhost` (иначе IPv6/IPv4 конфликт)
2. Правки в `server.js` требуют **перезапуска** (`node server.js`); правки в HTML/CSS/JS фронтенда — просто `Cmd+R`
3. Сквозная нумерация страниц между главами считается **в реальном видимом контейнере** (временно скрытом), не в отдельном клоне — иначе нумерация съезжает из-за тонких различий условий измерения
4. `document.fonts.ready` обязателен перед любым расчётом разбивки на страницы
5. Разбивка текста на страницы — через **бинарный поиск** (не по одному слову) — для скорости
6. Кэш количества страниц **по каждой главе отдельно** в `sessionStorage` — не пересчитывать одно и то же при разных переходах
7. Предзагрузка текста/картинок соседних глав в фоне
8. **Закладки** хранят позицию **по словам** (`word_offset`), а не номер страницы — номер страницы зависит от размера экрана читателя
9. SVG-иллюстрации делались через OpenCV (трассировка/кластеризация цветов) — в среде нет интернета для potrace/vtracer
10. Большие "внешние" SVG (под 700КБ) оптимизированы через растрирование + повторную трассировку с разумным числом цветов

## Перед началом работы — обязательно
```bash
cd ~/Documents/2026/творчество26/site/my-book-site
npm install
# .env должен содержать: DB_USER, DB_PASSWORD, DB_NAME, DB_HOST=127.0.0.1, SESSION_SECRET, ADMIN_TOKEN
node server.js
```

Если база создавалась давно — может не быть свежих колонок, накатить:
```bash
psql -d book_site -c "ALTER TABLE chapters ADD COLUMN IF NOT EXISTS epigraph TEXT;"
psql -d book_site -c "ALTER TABLE chapters ADD COLUMN IF NOT EXISTS illustration VARCHAR(255);"
psql -d book_site -c "ALTER TABLE reading_progress ADD COLUMN IF NOT EXISTS word_offset INTEGER DEFAULT 0;"
```
