# my-book-site — проект для публикации романов

## 🧪 Как запустить локально

```bash
npm install
cp .env.example .env
```

Откройте `.env` и заполните реальными значениями:
- данные подключения к PostgreSQL (`DB_USER`, `DB_PASSWORD`, `DB_NAME`...)
- `SESSION_SECRET` — любая длинная случайная строка
- `ADMIN_TOKEN` — придумайте свой токен для доступа к админке

Создайте базу данных и таблицы:
```bash
createdb your_db_name
psql -U postgres -d your_db_name -f db/schema.sql
```

Запустите сервер:
```bash
node server.js
```

Откройте `http://localhost:3000`

## 📄 Страницы сайта

| Страница | Назначение |
|---|---|
| `/index.html` | Главная страница |
| `/registration.html` | Регистрация / вход читателя |
| `/admin.html` | **Админка** — добавление книг и глав (вход по `ADMIN_TOKEN` из `.env`) |
| `/book.html?book=slug&chapter=1` | Чтение конкретной главы книги (флипбук) |
| `/book.html` (без параметров) | Старый режим — читает текст из `localStorage` (для совместимости с `send_text.html`, если он у вас остался) |

## ⚙️ Как опубликовать роман — два способа

### Способ 1: через админку (для коротких правок и небольших глав)
1. Откройте `/admin.html`, введите `ADMIN_TOKEN`
2. Заполните форму «Добавить книгу» — один раз для каждого романа
3. В форме «Добавить главу» выберите книгу, впишите номер главы и вставьте текст
4. Нажмите «Опубликовать главу»

### Способ 2: импорт готового .docx файла — одна глава
```bash
node import-docx.js <slug-книги> <номер-главы> <путь-к-файлу.docx> ["Заголовок главы"]
```
Пример:
```bash
node import-docx.js temny-voskhod 1 ./romans/glava1.docx "Глава первая"
```

### Способ 3: массовая загрузка ВСЕЙ книги сразу

**🌐 Через браузер (без терминала) — самый простой способ:**

Откройте `/admin.html` → раздел **«Загрузить книгу целиком (.docx)»**:
1. Выберите книгу из списка
2. Выберите файл романа (.docx)
3. Выберите режим:
   - **«По списку заголовков»** (надёжнее) — дополнительно загрузите файл `toc.txt` со списком заголовков глав, по одному на строку, точно как в самом романе
   - **«Автоматическое распознавание»** — без файла содержания, скрипт сам ищет «Глава 1», «Глава первая» и т.п.
4. Нажмите «Загрузить и разбить на главы»

**💻 Через терминал (`import-book.js`)** — то же самое, но из командной строки:

Подходит, если у вас уже написан весь роман и нужно опубликовать все главы одной командой.

**🏆 Самый надёжный способ — со списком заголовков (рекомендуется):**

Подготовьте файл `toc.txt` — список заголовков глав, по одному на строку, **точно как они написаны в самом романе** (см. пример в `romans-example/toc-example.txt`):
```
Глава 1. Название первой главы
Глава 2. Название второй главы
Глава 3. Название третьей главы
```

Затем:
```bash
# Сначала проверка без записи в базу:
node import-book.js temny-voskhod --file ./romans/temny-voskhod.docx --toc ./romans/toc.txt --preview

# Если разбивка верна — публикуем:
node import-book.js temny-voskhod --file ./romans/temny-voskhod.docx --toc ./romans/toc.txt
```
Скрипт находит каждый заголовок прямо в тексте романа и берёт весь текст между соседними заголовками как текст главы. Надёжнее автоматического угадывания — подходит для любого формата нумерации, даже если он отличается от книги к книге.

⚠️ Если заголовок не найден — скрипт остановится и покажет, какой именно (проверьте точное совпадение текста — опечатки, лишние пробелы).

**Если главы — отдельные файлы .docx:**
```bash
# Сначала проверка без записи в базу:
node import-book.js temny-voskhod --folder ./romans/temny-voskhod --preview

# Если разбивка верна — публикуем:
node import-book.js temny-voskhod --folder ./romans/temny-voskhod
```
Номер главы скрипт берёт из цифр в названии файла (`глава-1.docx`, `ch02.docx`, `03.docx` — все варианты подойдут).

**Если весь роман в одном .docx файле, без отдельного toc.txt (авто-угадывание):**
```bash
node import-book.js nemaya-rybka --file ./romans/nemaya-rybka.docx --preview
node import-book.js nemaya-rybka --file ./romans/nemaya-rybka.docx
```
⚠️ Менее надёжно — если автораспознавание не сработало, используйте способ со списком заголовков (`--toc`) выше.

⚠️ Во всех способах книга с этим `slug` должна быть уже создана (через `/admin.html` или вручную SQL-запросом).

## 📑 Содержание книги

Список всех опубликованных глав со ссылками на чтение:
```
http://localhost:3000/contents.html?book=temny-voskhod
```
На странице чтения главы (`book.html`) есть кнопка **«← Содержание»** в правом верхнем углу — ведёт обратно к этому списку.

После публикации глава доступна по адресу:
```
http://localhost:3000/book.html?book=temny-voskhod&chapter=1
```

## 🔖 Закладки для читателей

На сайте теперь есть вход (`/registration.html` — вкладки «Войти» / «Регистрация»). Авторизованный читатель видит на странице главы кнопку **«🔖 Сохранить закладку»** — она запоминает книгу, главу и **позицию по словам** в тексте (а не номер страницы — он зависит от размера экрана и каждый раз пересчитывается по-разному).

На странице «Содержание» (`contents.html`) у залогиненного читателя с сохранённой закладкой появляется кнопка **«📖 Продолжить чтение»** — открывает книгу точно с того места, где читатель остановился, независимо от размера его экрана.

## ✅ Что уже реализовано

- Express-сервер с PostgreSQL (регистрация, вход, закладки — старая система через `users.zakladka`)
- Полноценные маршруты для книг и глав (`/api/books`, `/api/books/:slug`, `/api/books/:slug/chapters/:num`)
- Защищённая админка для добавления контента (`requireAdmin` + токен из `.env`)
- Страница чтения — флипбук с анимацией перелистывания
- **Автоматический пересчёт страниц при изменении размера окна** (без нужды нажимать F5)
- Вставка картинок в текст книги — **с сохранением картинок при resize**
- Конвертер `.docx → текст` для быстрой загрузки готовых романов (по одной главе или сразу всей книгой с авто-разбивкой)
- Страница «Содержание» со списком глав книги (`contents.html?book=slug`)
